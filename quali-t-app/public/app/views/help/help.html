Quality Attributes, Catalogs, Projects?

Before you can benefit from QUALI-T you must understand the key models of the application. There are 5 of them:

Key Model Explaination
Quality Attribute Template (QA Template) Create templates for quality attributes, which can be then used while creating of a new project. The big advantage of templates is the option to add variables.
Catalog Group multiple quality attribute templates into one catalog and re-use the catalog for creating new projects. Every project of the same catalog has automatically all quality attribute templates inside.
Project This is the most important element in QUALI-T, because it is the original intention of the software. Specify quality attributes for the project you are working on. A project consists of a set of quality attributes instances.
Quality Attribute Instance (QA) Quality Attribute Instances live inside projects. After creating the project based on a catalog, each of the quality attribute template gets an instance. The instance is now decoupled from the catalog and can be edited. The user wants typically to change values of the variables, change quality property states or export to Issue Tracking System (JIRA).
Quality Properties (QP) Quality Properties are useful for measuring some specific properties of a quality attribute. QP's are boolean values and can either be satisfied or not. An example could be S - Specific. Each quality attribute has a checkbox next to it. If the quality property is specific, the checkbox could be checked. This means that the S - Specific quality property is satisfied.

FAQ

Question: We have a new project for our customer and I'm new to QUALI-T. How can I benefit from this software?

Answer: Follow the steps below.


Step Instructions
1. Create Customer First you need to create the customer in the QUALI-T system.
2. Check existing catalogs & quality attributes Maybe there is already a matching catalog with quality attributes. Check the list of catalogs.
3. Create a new catalog, if no matching catalog was found. Create desired catalog with quality attribute templates. Specify variables for quality attribute templates.
4. Create Project by using a catalog Specify your projects properties (name, JIRA connection, quality properties) and choose the matching catalog.
5. Edit Project After creating the project you will be in the edit mode. Specify each quality attribute instance by filling out the variables.
6. Administrate the project in the future Keep in mind that you have also an overview of all quality attribute instances in the project. You can set the state of quality properties, export to Issue Management System, check for validity (fuzziness detector API).
