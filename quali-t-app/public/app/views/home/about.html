Bachelor Thesis

QUALI-T is the webapplication which was designed and built during the Bachelor Thesis of Science at HSR (Hochschule für Technik, Rapperswil).

Idea behind

Software projects are often specified very detailed. This is good. But there is no tool where you can specify non-functional requirements (also called quality attributes).

This was the begin of QUALI-T. We started to analyse non-functional requirements, and specified a way to put them into our application. The main advantage is that everything is specified in one place, where others can also access.

Because all data is stored in the same application, there is a new opportunity of providing information in relation to other projects. These are statistics like the most used value or the average vale (for numbers).

With the connection to the Issue and Project Management Tool JIRA and the possibility to create Exports (PDF, JSON, XML), this tool helps you to make better software from the begin.

There is also a sub-project where you can create new plug-in like detectors to verifiy a project's quality attributes. Check it out here: Fuzziness Detector API on Github.
